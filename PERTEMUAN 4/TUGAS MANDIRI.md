1. Apa yang Saya Pahami tentang Naming Convention?
   - Menurut saya, Naming Convention adalah aturan atau kebiasaan dalam memberi nama pada variabel, class, dan bagian lain dalam program.
     Tujuannya supaya kode yang kita buat lebih rapi dan gampang dibaca. Di Java, biasanya nama class diawali huruf besar, sedangkan nama variabel diawali huruf kecil.
     Jika kita mengikuti aturan ini, program jadi terlihat lebih terstruktur dan tidak membingungkan.
3. Mengapa Penamaan Variabel dan Class Itu Penting?
   - Penamaan itu penting karena bisa mempengaruhi seberapa mudah orang memahami program yang kita buat.
     Jika nama variabelnya jelas misalnya totalHarga, orang lain langsung tahu bahwa itu merupakan total harga.
     Berbeda jika nama variabelnya x, a1, atau data123, orang lain tidak akan tahu itu digunakan untuk apa.
     Tidak hanya menyusahkan orang lain, hal itu juga akan menyusahkan diri sendiri.
3. Bagaimana Operator Aritmatika Bekerja dalam Java?
   - Operator aritmatika di Java digunakan untuk melakukan perhitungan matematika seperti tambah, kurang, kali, dan bagi.
     "+" untuk pertambahan, "-" untuk pengurangan, "*" untuk perkalian, "/" untuk pembagian, "%" untuk sisa bagi.
5. Contoh yang Saya Pahami dari Latihan
  - Dari latihan yang saya kerjakan, saya jadi lebih paham tentang naming convention. Saya juga tahu kalau nama file harus sama dengan nama class supaya program bisa dijalankan.

    Dengan mengikuti aturan ini, kode jadi lebih rapi dan gampang dibaca. Saya juga menyadari kalau cara menampilkan output di Java berbeda dengan C++.
